# 📱 手机访问项目配置指南

## 方案一：直接使用服务器 IP 地址（推荐）⭐

### 1. 获取服务器 IP 地址

```bash
# 查看服务器 IP
hostname -I | awk '{print $1}'
```

**当前服务器 IP**：`9.129.127.116`

### 2. 检查服务监听状态

```bash
# 检查端口是否监听在所有网络接口
netstat -tlnp | grep -E ':(5000|9091)'
```

预期输出：
```
tcp6       0      0 :::9091                 :::*                    LISTEN
tcp6       0      0 :::5000                 :::*                    LISTEN
```

注意：`:::` 表示监听在所有网络接口（IPv6），包括外部访问。

### 3. 手机访问地址

确保手机和服务器在同一个网络（或服务器允许外部访问）：

- **前端访问**：`http://9.129.127.116:5000`
- **后端 API**：`http://9.129.127.116:9091/api/v1`

### 4. 配置前端 API 地址

**重要**：前端需要配置正确的后端 API 地址才能正常工作。

查看前端的环境变量：
```bash
grep EXPO_PUBLIC_BACKEND_BASE_URL /workspace/projects/.cozeproj/scripts/dev_run.sh
```

如果配置的是 `localhost`，需要修改为服务器 IP：

**方案 A：修改环境变量**
```bash
# 在终端中设置环境变量
export EXPO_PUBLIC_BACKEND_BASE_URL="http://9.129.127.116:9091"

# 然后启动前端
cd /workspace/projects/client
pnpm run web
```

**方案 B：创建启动脚本**
创建 `mobile-access.sh`：
```bash
#!/bin/bash
export EXPO_PUBLIC_BACKEND_BASE_URL="http://9.129.127.116:9091"
cd /workspace/projects/client
pnpm run web
```

### 5. 测试访问

从手机浏览器访问：
```
http://9.129.127.116:5000
```

## 方案二：SSH 端口转发（适用于本地开发）

如果服务器不允许外部访问，可以使用 SSH 端口转发。

### 1. 在手机端运行 SSH 端口转发

**iOS（使用 Termius 或 Blink）**：
```
ssh -L 5000:localhost:5000 -L 9091:localhost:9091 user@9.129.127.116
```

**Android（使用 Termux）**：
```bash
pkg install openssh
ssh -L 5000:localhost:5000 -L 9091:localhost:9091 user@9.129.127.116
```

### 2. 手机访问地址

```
http://localhost:5000
```

## 方案三：使用 ngrok 创建公网地址（最简单）⭐⭐

### 1. 安装 ngrok

```bash
# 下载 ngrok
wget https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-linux-amd64.zip
unzip ngrok-v3-stable-linux-amd64.zip
sudo mv ngrok /usr/local/bin/

# 登录 ngrok（需要注册账号）
ngrok config add-authtoken YOUR_AUTH_TOKEN
```

### 2. 创建公网隧道

**方案 A：分别创建两个隧道**
```bash
# 前端隧道（端口 5000）
ngrok http 5000 &

# 后端隧道（端口 9091）
ngrok http 9091 &
```

**方案 B：创建一个隧道同时转发两个端口**
使用 ngrok 配置文件 `ngrok.yml`：
```yaml
tunnels:
  frontend:
    proto: http
    addr: 5000
  backend:
    proto: http
    addr: 9091
```

然后运行：
```bash
ngrok start frontend backend
```

### 3. 获取公网地址

ngrok 会显示公网地址，类似：
```
Frontend: https://xxxx-xx-xx-xx-xx.ngrok-free.app
Backend:  https://yyyy-yy-yy-yy-yy.ngrok-free.app
```

### 4. 配置前端使用 ngrok 地址

```bash
export EXPO_PUBLIC_BACKEND_BASE_URL="https://yyyy-yy-yy-yy-yy.ngrok-free.app"
```

### 5. 手机访问

使用 ngrok 提供的公网地址：
```
https://xxxx-xx-xx-xx-xx.ngrok-free.app
```

## 🔧 快速启动脚本（推荐使用）

### 创建移动端访问脚本

```bash
#!/bin/bash

echo "=========================================="
echo "📱 启动移动端可访问的服务"
echo "=========================================="
echo ""

# 获取服务器 IP
SERVER_IP=$(hostname -I | awk '{print $1}')
BACKEND_URL="http://${SERVER_IP}:9091"

echo "服务器 IP: $SERVER_IP"
echo "后端 API: $BACKEND_URL"
echo ""

# 设置环境变量
export EXPO_PUBLIC_BACKEND_BASE_URL="$BACKEND_URL"

# 启动前端
echo "正在启动前端服务（端口 5000）..."
echo "手机访问地址：http://${SERVER_IP}:5000"
echo ""

cd /workspace/projects/client
pnpm run web
```

保存为 `mobile-access.sh`，然后运行：
```bash
chmod +x mobile-access.sh
./mobile-access.sh
```

## ⚠️ 常见问题

### 问题1：手机无法访问服务器 IP

**原因**：
- 防火墙阻止了外部访问
- 手机和服务器不在同一网络
- 云服务器需要配置安全组

**解决方案**：
1. 检查防火墙规则
2. 如果是云服务器，配置安全组开放端口 5000 和 9091
3. 使用 ngrok 方案

### 问题2：前端请求后端失败

**原因**：
- 前端配置的后端地址是 localhost
- 手机无法访问 localhost

**解决方案**：
必须修改 `EXPO_PUBLIC_BACKEND_BASE_URL` 为服务器 IP 或公网地址。

### 问题3：ngrok 隧道不稳定

**原因**：
- 免费版 ngrok 隧道地址会变化
- 连接超时

**解决方案**：
- 使用付费版 ngrok 固定域名
- 或者每次演示时重新创建隧道

## 🎯 推荐方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **直接 IP 访问** | 稳定、快速 | 需要网络配置 | 内网演示、服务器允许外部访问 |
| **SSH 端口转发** | 安全 | 需要手机端 SSH 客户端 | 本地开发测试 |
| **ngrok 公网隧道** | 最简单、无需配置 | 免费版不稳定、地址变化 | 临时演示、远程访问 |
| **Expo Go** | 真机测试 | 需要安装 Expo Go | 开发阶段测试 |
